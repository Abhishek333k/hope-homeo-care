## ⌨️ 1. Keyboard Shortcuts & Keystrokes

The project implements several "invisible" power-user features to enhance navigation and accessibility.

### A. Global Navigation Shortcuts
| Key | Context | Action |
| :--- | :--- | :--- |
| **`Escape`** | Any Modal Open | Instantly closes the active modal (Booking, Gallery, or Login). Resets modal state to default. |
| **`ArrowRight`** | Gallery Slider | Navigates to the **Next** image in the interactive gallery. |
| **`ArrowLeft`** | Gallery Slider | Navigates to the **Previous** image in the interactive gallery. |

### B. Input Field Keystroke Masking
*   **Target Field:** `#patient-phone` / `#login-phone`
*   **Behavior:** A real-time `input` listener sanitizes every keystroke. 
*   **Result:** It physically prevents the entry of non-numeric characters and automatically truncates the input at 10 digits. This prevents database "dirty data" errors.

## 🔔 2. Alert & Notification System (Toast UI)

Instead of browser-native `alert()` boxes (which are intrusive), the project uses a custom **Toast Notification System** (`window.showToast`) for a non-blocking, premium experience.

### A. Display Archetypes
*   **Success Archetype (`type='success'`):** 
    *   **Visual:** Deep Slate-800 background, white text, checkmark icon.
    *   **Trigger:** Successful appointment booking, OTP delivery, or login verification.
*   **Error Archetype (`type='error'`):**
    *   **Visual:** Rose-500 background, white text, alert icon.
    *   **Trigger:** Validation failures, network timeouts, or rate-limiting (spam protection).

### B. Trigger Matrix (Detailed)
| Trigger Event | Type | Message Context |
| :--- | :--- | :--- |
| **Empty Form Fields** | `error` | "Please fill out all fields and accept the legal disclaimer." |
| **Invalid Phone** | `error` | "Please enter a valid 10-digit mobile number." |
| **Spam Protection** | `error` | "Please wait a minute before submitting another request." |
| **Booking Success** | `success` | "Appointment requested successfully!" |
| **Network Failure** | `error` | "Network error: Could not send request..." |
| **Auth Flow** | `success` | "Login successful!" / "OTP sent to your phone" |

### C. Technical Implementation Status
*   **Duration:** 4,000ms (Auto-dismisses).
*   **Z-Order:** Hard-coded to `z-[100]` ensuring it always displays above modals (`z-50`) and headers (`z-40`).
*   **Animation:** Uses CSS transitions for a smooth "Slide Up" (`translate-y-10` to `translate-y-0`) and "Fade In" effect.
